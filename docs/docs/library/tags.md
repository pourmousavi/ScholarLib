# Tags

Tags are labels for grouping **semantically similar documents** together. Use tags when papers share a common theme, methodology, topic, or characteristic. Unlike folders, a document can have multiple tags, enabling cross-cutting organization.

## What Tags Are For

Think of tags as a way to mark papers that belong together conceptually. A tag like "Deep Learning" groups all papers that use or discuss deep learning methods, regardless of which folder they're stored in. Similarly, a "Review Papers" tag identifies all literature reviews across your library.

- **Semantic grouping** — Group papers by what they're about, not where they're stored
- **Cross-folder organization** — A paper can have multiple tags without duplication
- **Quick filtering** — Click a tag to instantly see all documents with that tag
- **AI Chat scope** — Ask AI questions about all papers with specific tags
- **Shareable** — Share tags with collaborators (papers with that tag become visible to them)

## Tags vs Collections

Tags and Collections work together but serve different purposes. See the [Collections](/library/collections) section for a detailed comparison.

| Aspect | Tags | Collections |
|--------|------|-------------|
| Purpose | Group similar documents | Organize for a specific goal |
| Scope | Semantic/topical | Project/task-oriented |
| Contains | Documents directly | Tags (documents indirectly) |
| Example | "Machine Learning", "Survey" | "PhD Thesis Chapter 3" |

## Creating Tags

Tags are created in the **TAGS** section of the left sidebar:

1. Look for the **TAGS** section below your folders
2. Click the **+** button next to "TAGS"
3. Enter a name for your tag and press Enter
4. The tag is created with an automatic color

## Managing Tags

Right-click any tag in the TAGS section to access options:

- **Edit tag** — Change the name, color, category, or add a description
- **Filter by this tag** — Show all documents with this tag
- **Share tag** — Make this tag and its documents visible to collaborators
- **Add to collection** — Include this tag in a collection

## Assigning Tags to Documents

- **Right-click menu:** Right-click a document → "Manage tags..." → Check/uncheck tags
- **Edit metadata:** Right-click → "Edit metadata" → Add tags in the Tags field
- **Bulk tagging:** Select multiple documents, then use bulk actions to add/remove tags

## Filtering by Tags

- **Single tag:** Click a tag to see all documents with that tag
- **Multiple tags:** Hold Shift and click to select multiple tags
- **AND mode:** Show documents that have ALL selected tags
- **OR mode:** Show documents that have ANY of the selected tags

## Merging Tags

If you have duplicate or similar tags:

1. Click the merge icon next to the TAGS header
2. Select source tags (will be deleted)
3. Select target tag (will be kept)
4. Click "Merge Tags" — all documents are updated automatically

## Keywords vs Tags

- **Keywords** — Automatically extracted from the paper's metadata (author-assigned)
- **Tags** — Your personal organizational labels that you create and assign

Both can be displayed on document cards. Control this in Settings → Appearance.

## Tips for Effective Tagging

:::tip Best Practices
- Keep tag names short and consistent
- Use tags for *what a paper is about*, not *what you're using it for* (use collections for that)
- Consider tags like "To Read", "Reviewed", or "Key Paper" for workflow
- Don't over-tag — 2-5 tags per document is usually sufficient
:::
